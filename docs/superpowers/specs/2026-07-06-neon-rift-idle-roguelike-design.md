**작성일:** 2026-07-06
**프로젝트 위치:** `d:\node\webservice\webgame` (gbi-node·lotto-ai와 완전 분리된 신규 프로젝트)

## 2. 아키텍처 개요

정적 클라이언트 전용 게임. 서버·빌드 단계 없음. `index.html`이 `js/main.js`(ES 모듈)를 로드하고, 모듈이 게임 루프·상태·렌더를 구동한다. 모든 상태는 메모리(런) + localStorage(메타).

**핵심 원칙 — 데이터 주도(Data-Driven):** 캐릭터·스킬·적·보스·바이옴·메타 업그레이드를 `js/data/*.js`의 순수 데이터 테이블로 정의한다. 런타임 시스템은 이 데이터를 *해석*할 뿐, 콘텐츠를 하드코딩하지 않는다. → 콘텐츠 추가 = 데이터 항목 추가(코드 변경 최소).

**로직/표현 분리(테스트 가능성):** 순수 로직 모듈(스탯 계산·스킬 스케일링·메타 비용·RNG·저장)은 DOM/Canvas/WebAudio에 의존하지 않아 Node에서 `node --test`로 유닛 테스트한다. 표현 모듈(render·audio·input·ui)은 브라우저 API에 의존하며 수동/시각 검증한다.

### 게임 루프
- 고정 타임스텝 로직 업데이트(60Hz, 누적자 방식) + `requestAnimationFrame` 렌더.
- 델타타임 기반 이동/쿨다운. 탭 비활성 시 과도한 catch-up 방지(최대 스텝 클램프).
- 씬(scene) 전환: `title → loadout → run → gameover → meta` (+ 오버레이: levelup, settings, pause).

### 상태 모델
- `metaState` (영속, localStorage): 소울 잔액, 구매한 메타 업그레이드 레벨, 해금된 캐릭터·스킬, 설정(볼륨·뮤트·오토파일럿), 최고기록.
- `runState` (휘발): 현재 캐릭터, 보유 스킬·레벨, 파생 스탯, 시간·스테이지·XP·레벨·골드, 엔티티 풀, RNG 시드.

## 4. 게임 시스템 상세

### 4.1 스탯 시스템 (`stats.js`)
파생 스탯 = 캐릭터 기본치 × (1 + 메타 보너스 + 런 픽 보너스) 형태로 계산.
표준 스탯 키: `maxHp, hpRegen, damage, atkSpeed(공격간격 배수), area, projectiles, pierce, crit, critMult, moveSpeed, pickupRange, cooldown, goldGain, soulGain, xpGain`.
- 합연산(+flat)과 곱연산(×%)을 분리해 누적. 최종 스냅샷을 `runState.stats`에 캐시하고 픽/레벨업 시 재계산.

### 4.2 스킬 시스템 (`skills.js` + `skillScaling.js` + `data/skills.js`)
스킬 데이터 스키마(예):
```js
{
  id: 'magic_bolt', name: '매직 볼트', type: 'projectile',
  tags: ['arcane'], base: { damage: 8, count: 1, speed: 7, cooldown: 40, pierce: 0 },
  scale: { damage: +4, count: [3,5,7], cooldown: -3 },   // 레벨별 증가 규칙
  maxLevel: 8,
  evolveInto: 'arcane_storm', evolveReq: { passive: 'multi_cast', level: 8 }, // 시너지 진화
}
```
스킬 타입(런타임 동작): `projectile`(직선/조준), `orbital`(회전 궤도), `aura`(주변 지속 장판), `beam`(관통 빔), `chain`(연쇄), `summon`(포탑/사역마), `passive`(스탯 강화). 각 타입은 `skills.js`에 실행 함수 하나로 구현되고, 데이터로 파라미터화된다.

### 4.3 레벨업/픽 (`levelup.js`)
- 적 처치 → XP젬 드롭 → 획득 시 XP 누적 → 임계치 도달 시 레벨업.
- 레벨업마다 **3택1** 후보 생성: (a) 미보유 스킬 신규, (b) 보유 스킬 +1레벨, (c) 패시브. 후보 풀은 해금 상태·중복·최대레벨·진화 가능 여부로 필터링, 가중 랜덤(RNG 시드).
- 진화: 특정 스킬이 최대레벨 + 요구 패시브 보유 시 후보에 "진화" 등장 → 강화판 스킬로 교체.

### 4.4 전투·엔티티 (`combat.js`, `entities.js`, `spawner.js`)
- 오브젝트 풀로 GC 최소화(적·투사체·파티클·플로팅텍스트·픽업).
- 충돌: 투사체-적 원-원 판정. 데미지 적용→크리 판정→상태이상 부여→데미지 숫자·히트플래시·파티클.
- 스포너/디렉터: 경과 시간으로 스폰율·적 HP/속도 스케일↑. 바이옴별 적 세트에서 가중 선택. N초/스테이지마다 보스. 무한 스케일링.

### 4.5 메타 성장 (`meta.js`, `data/metaUpgrades.js`)
- 런 종료 시 소울 = f(도달 스테이지, 생존 시간, 처치 수) 정산.
- 상점: 영구 업그레이드(예: 시작 체력·시작 공격력·골드획득·소울획득·리롤 횟수·부활 1회). 비용은 레벨별 증가 곡선, 최대레벨 존재.
- 해금: 소울/업적 조건으로 캐릭터·스킬 언락 → 로드아웃 선택지·드롭 풀에 추가.

### 4.6 오디오 (`core/audio.js`)
- **SFX:** ZzFX풍 경량 신스를 재구현(MIT 라이선스 아이디어 기반, 자체 작성). 이벤트: `shoot, hit, crit, kill, coin, levelup, pick, boss, hurt, death, ui, upgrade`.
- **BGM:** `assets/bgm/`의 파일이 있으면 로드·루프·볼륨·뮤트. 없으면 Web Audio 프로시저럴 칩튠 루프로 폴백 → 파일 없이도 항상 음악.
- 설정: 마스터·SFX·BGM 볼륨 + 뮤트, localStorage 저장. 최초 사용자 제스처에서 AudioContext resume(브라우저 정책).

### 4.7 입력/오토파일럿 (`core/input.js`)
- 기본 오토파일럿: 위협 회피 + 픽업 유도 조향 벡터 계산(가장 위험한 적 반대 + 근접 XP젬 방향 가중).
- 수동 토글: 포인터 드래그(모바일) / WASD·방향키(데스크톱)로 직접 이동. 공격은 항상 자동 조준.

## 5. v1 콘텐츠 목표 (풍성한 출시)

데이터 항목으로 정의. 아래는 v1 최소 목표치이며 데이터 추가로 확장 가능.

- **캐릭터 4종:** 검사(회전베기·근접·고HP), 마법사(투사체·광역), 궁수(관통·직선), 정령술사(오라/소환·지속). 각자 시작 스킬 1 + 고유 패시브 1.
- **스킬 20+종:** 투사체 4, 오비탈 2, 오라 3, 빔 2, 체인 2, 소환 2, 패시브 6+, 진화 4쌍 이상.
- **적 6종 + 엘리트 변형:** 스워머·러너·탱커·슈터·분열체·돌진체.
- **보스 3종:** 바이옴별 1, 페이즈·패턴 보유.
- **바이옴 2~3:** 팔레트+적 세트+보스+난이도곡선. 무한 스케일링.
- **메타 업그레이드 6~8 + 해금:** 캐릭터 3종·스킬 다수를 소울로 해금.

## 7. 테스트 전략

- **유닛(`node --test`, 무빌드):** `stats.js`(파생 스탯 합·곱 누적), `skillScaling.js`(레벨 스케일·진화 판정), `meta.js`(비용 곡선·구매 가능·해금), `rng.js`(시드 결정성·재현), `storage.js`(저장/로드·마이그레이션·기본값).

## 8. 배포

## 9. 제작 순서 (수직 슬라이스 우선)

1. **엔진 코어 수직 슬라이스** — 루프+엔티티 풀+1캐릭+시작스킬 1+투사체+적 1종+충돌/데미지+XP/레벨업 3택1+HUD. "돌아가는 재미" 확보.
2. **스킬 시스템 확장** — 스킬 타입 전 종류 + 데이터 스킬 20+ + 진화.
3. **적·보스·바이옴** — 스포너 디렉터 + 적 6종 + 보스 3 + 바이옴 2~3.
4. **메타 성장** — 소울 정산 + 상점 + 해금 + 로드아웃 화면 + localStorage 저장.
5. **아트 폴리시** — 네온 이펙트·스크린셰이크·데미지숫자·파티클·UI 테마.
6. **사운드** — SFX 신스 + BGM 폴백/파일 + 설정.
7. **참조 문서·CLAUDE.md·테스트 마감.**

각 단계는 독립적으로 플레이 가능한 산출물을 남긴다.
