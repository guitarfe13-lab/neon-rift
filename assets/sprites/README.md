# 캐릭터·몬스터 스프라이트(사람/몬스터 이미지) 넣는 법

이 폴더에 **투명 배경 PNG**를 넣으면 게임이 자동으로 그 이미지를 캐릭터/몬스터로 사용합니다.
**없는 파일은 지금의 네온 코드 스프라이트로 자동 폴백**되므로, 하나씩 넣어도 안 깨집니다.

## 이미지 규격
- **형식**: PNG, **투명 배경**(알파). 배경색·박스 없이 캐릭터만.
- **구도**: **정면(카메라를 바라보는) 전신 1장**. 아래쪽이 발. 좌우 이동 시 게임이 자동으로 좌우 반전합니다.
- **크기**: 정사각에 가깝게. 캐릭터·몬스터 **256×256**, 보스 **512×512** 권장(고해상도로 넣으면 게임에서 자동 축소).
- **여백**: 캐릭터가 프레임을 꽉 채우되 살짝 여백. 그림자·글로우는 게임이 별도 처리하니 굽지 말 것(선택).

## 어디서 만드나 (사람/몬스터)
- **AI 이미지 생성**(권장): "정면 전신, 투명 배경, 게임 캐릭터, <설명>" 프롬프트로 생성 후 배경 제거(remove.bg 등). 예: 미드저니/DALL·E/Stable Diffusion/니지저니.
- **무료 CC0 에셋**: itch.io, OpenGameArt(CC0), Kenney — "top-down character", "monster sprite" 검색. 라이선스 확인.

## 스프라이트시트 애니메이션 (지원됨)
한 이미지에 여러 포즈(프레임)를 **균일 격자**로 배치한 시트로 대기/공격 애니메이션을 넣을 수 있습니다.

### 파일
- **`assets/sprites/<id>_sheet.png`** (또는 `.svg`). 예: `blade_sheet.png`.
  - 단일 이미지 `<id>.png`와 별개. **시트가 있으면 시트가 우선**, 없으면 단일 이미지 → 코드 스프라이트로 폴백.
- **균일 격자**: 모든 칸이 같은 크기, 각 칸에 캐릭터가 비슷한 위치/크기로 중앙정렬돼야 깔끔합니다.

### 설정 (`js/data/characters.js`의 캐릭터에 `sheet`)
```js
sheet: { cols:4, rows:3, fps:7, anims:{ idle:[0,11], attack:[1,2,6] } }
```
- `cols`/`rows`: 격자 열·행 수. 게임이 이미지를 이 격자로 자동 분할.
- **프레임 번호 = 행×cols + 열** (0부터, 좌→우, 위→아래). 4×3이면 0~11.
- `fps`: 프레임 전환 속도. `anims.idle`=평상시 루프, `anims.attack`=스킬 발동 시 재생.
- (선택) `scale`: 시트 전용 크기 보정.

### 예: 올려주신 기사 시트(4열×3행, 12포즈)
- 파일명을 **`blade_sheet.png`** 로 저장 → `assets/sprites/`에 넣기.
- 검사 설정은 이미 `sheet:{cols:4,rows:3,fps:7,anims:{idle:[0,11],attack:[1,2,6]}}`로 되어 있습니다.
- 프레임 번호가 실제 포즈와 어긋나면 `idle`/`attack` 배열 숫자만 바꾸면 됩니다(예: 공격을 `[1,2,4,6]`으로).
> 포즈들이 칸마다 크기·위치가 제각각이면 잘릴 수 있습니다 → 가능하면 **균일 격자**(칸마다 캐릭터 중앙정렬)로 만들면 가장 깔끔합니다.
